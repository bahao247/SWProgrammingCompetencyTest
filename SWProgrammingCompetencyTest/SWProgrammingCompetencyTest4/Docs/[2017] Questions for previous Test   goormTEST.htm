[2017/1st]Development of Self-driving Algorithm Submit completed

2Second30points

*Solution (click): C++, C, Java

Standard Library reference : C++, C, Java


VC Company's Self-driving Development Team is developing the new algorithm for self-driving.

This algorithm finds the shortest route from an origin to a destination while avoiding the parks displayed in the map.

In other words, it must find the shortest route to the destination only through the streets avoiding parks.



[Requirement]

City Data

1. The size is H * W. H is the height while W is the width (1≤H and W≤500). 

2. The city is organized of 1*1 sized grids. 

   Each grid contains a street or a park.

3. The origin position is always (0, 0) while the destination position is always (H-1, W-1).

4. The street is marked ‘.’ while the park is marked ‘X’. 

The origin position and the destination position are always street.


Constraint of Movement

1. Only the movement to the adjacent grid to the right, left, top, bottom is allowed, and an hour is needed to pass through a grid.

2. A passage through a park is not allowed. Only the streets can be passed through.

3. Moving out of the given map is not allowed.


Examples of Input


The following map shows the map of H=3 and W=3 in input 1.

The shortest path from the origin (0,0) to the destination (2,2) while meeting the constraint takes 4 hours.

The following diagram show one of the shortest paths that take 4 hours to reach the destination.



The following map shows the map of H=5 and W=4 in input 2.

Although the shortest path from the origin (0,0) to the destination (4,3) takes 7 hours as shown in the figure in the middle, it is not the answer since it fails to meet the constraint of not passing through the park.

  

Therefore, the shortest path from the origin to the destination while meeting the constraint takes 9 hours.




[Input Format]

In the first line, two integers H and W are input with a space as the separator.

H is the height while W is the width.

Beginning with the second line, a string of W characters are input in H lines.

The character string represents the grid data.


[Output Format]

Print the shortest path from the origin (0,0) to the destination (H-1,W-1).

Output -1 if the destination cannot be reached.
